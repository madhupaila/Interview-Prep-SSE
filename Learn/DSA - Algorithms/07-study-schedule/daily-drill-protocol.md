# Daily 15-Minute Drill Protocol

Do this **every study day before solving**. It builds the reflex to recognize patterns instantly — the single highest-leverage habit.

---

## The Drill (15 minutes)

### Part 1 — Flashcards (5 min)

Open [memory-map-flashcards.md](../01-patterns/memory-map-flashcards.md). Cover the answer columns.

- Read each **cue** aloud → say the **pattern** + **template anchor** in <5 seconds.
- Mark misses. Target **55/60**.

### Part 2 — Blank template (5 min)

On paper, reproduce **one** pattern's template from memory (rotate daily):

- Mon: Two Pointers + Sliding Window
- Tue: Binary Search (classic + on-answer)
- Wed: Tree DFS (bubble-up) + BFS (level loop)
- Thu: Graph BFS/DFS + Union-Find
- Fri: Backtracking (choose/explore/undo)
- Sat: DP (state + recurrence) + Kadane
- Sun: Heap (size-K) + Topo sort (Kahn)

Compare against the [pattern sheet](../01-patterns/00-pattern-master-index.md). Note gaps.

### Part 3 — Identify-only (5 min)

Read **3 random problem titles** (from the [company index](../06-company-question-index.md) or anywhere). For each, **don't solve** — just say:

1. The **pattern** (and why, from the cue)
2. The **target complexity** (from constraints)
3. The **template** you'd start from

---

## Weekly Recognition Test

Once a week, take 10 unseen problem statements and classify them in 5 minutes total. Score yourself:

| Score | Meaning |
|-------|---------|
| 9–10 | Pattern recognition is interview-ready |
| 7–8 | Solid; drill the missed families |
| < 7 | Re-read those pattern sheets + cue dictionary |

---

## Why This Works

Interviews reward **fast, correct mapping** from problem → pattern → template. Solving is the easy part once you've matched the pattern. This drill trains the matching itself, separate from the slower act of coding.
