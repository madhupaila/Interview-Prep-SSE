# Senior React Interview Prep Guide

**Target:** 3–5 YOE Full Stack Engineer → **Senior Software Engineer** (Gen AI / LLM focus)

A structured, interview-ready React study system with **memory maps**, concept deep-dives, **80+ company-tagged questions**, and spoken answer scripts — the same learning style as the HLD/LLD tracks.

---

## How to Use This Repo

**Start here for confidence + memory mapping:**
1. [Structured Preparation & Memory Mapping](00-interview-framework/05-structured-preparation-and-memory-mapping.md) — **7 zones, MEMOP answer flow, 4-week schedule**
2. Print [One-Page Memory Poster](01-core-concepts/memory-map-visual-one-page.md) + drill [50 Flashcards](01-core-concepts/memory-map-flashcards.md) daily (15 min)
3. [Master Memory Map](01-core-concepts/memory-map-master.md) — full Z1–Z7 reference
4. [Decision Picker](00-interview-framework/04-decision-picker.md) — what hook/pattern when

**Then by week:**
1. **Week 1:** [Interview Framework](00-interview-framework/) + memorize maps
2. **Week 2:** [Core Concepts](01-core-concepts/) — rendering, state, hooks, performance
3. **Weeks 3–4:** [Classic React Questions](03-classic-react/questions/) (60) — speak answers aloud
4. **Week 5:** [Gen AI / LLM React](02-genai-llm-react/) — differentiator
5. **Week 6:** [Company Index](04-company-question-index.md) + timed mocks

**Per question flow:** Zone router → Memory hook → MEMOP answer → Code sketch → Tradeoffs

## Per-Question Template

Every question file includes:

1. Company tags + difficulty
2. **Memory hook** — one-line recall anchor
3. Concept being tested + why interviewers ask
4. **Options / approaches** table (what you could use)
5. **Decision picker** — what to use when (Senior pick)
6. Clarifying questions (if system design / build question)
7. Interview answer script (spoken walkthrough)
8. Code example (TypeScript + React 18+)
9. Tradeoffs table
10. Follow-up questions + how to answer
11. Related concepts + memory map link

---

## Senior React Interview Checklist

- [ ] Start with **mental model** (render → commit → paint), not API trivia
- [ ] Name **2–3 options**, then pick one with tradeoffs
- [ ] Mention **accessibility, performance, and error states** unprompted
- [ ] For GenAI UI: streaming, cancellation, optimistic UI, token/cost UX
- [ ] Tie answers to **real product** (ChatGPT UI, Notion AI sidebar, Copilot inline)
